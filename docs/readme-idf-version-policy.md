# README IDF Version Policy

## 1. 背景

项目已经明确采用 `ESP-IDF v5.5.4` 作为 `ESP32-P4` 当前阶段的固定基线，但入口文档此前没有把版本选择和未来升级条件写清楚。为了降低后续开发、构建和多 agent 协作中的歧义，需要把这部分信息固化到 `README.md`。

## 2. 已落地内容

本次已在 [README.md](/Users/andyhao/workspace/p4home/README.md) 中新增两部分说明：

- 当前 `ESP-IDF` 版本基线
- 未来何时考虑升级 `ESP-IDF`

## 3. 当前版本基线

当前项目固定使用：

- `ESP-IDF v5.5.4`

当前阶段默认不使用：

- `master`
- 漂移分支
- `v6.x`

所有 `firmware/` 相关构建与验证，默认都应先切换到 `v5.5.4`。

## 4. 升级触发条件

当前文档中约定，只有在以下情况下才考虑升级 `ESP-IDF`：

- `M1` 到 `M3` 的板级 bring-up、`LVGL`、`ESP-SR` 基础链路已经稳定
- 当前版本在 `ESP32-P4` 支持、驱动或组件兼容性上出现明确阻塞
- 新版本带来项目明确需要的能力，而不是仅仅因为“有新版本”

升级前仍需额外验证：

- 独立验证新版本环境
- 确认不会破坏现有 `firmware/` 构建
- 确认不会破坏已完成功能

## 5. 结果

这次改动让项目入口文档与当前实际环境保持一致，并把未来升级 `ESP-IDF` 的判断标准前置，减少后续上下文重复讨论。
