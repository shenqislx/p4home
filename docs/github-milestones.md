## 3. 当前设置

- 所有 milestones 当前均为 `open`
- 当前未设置 `due date`
- milestone 名称与 [project-milestones.md](/Users/andyhao/workspace/p4home/docs/project-milestones.md) 保持一致

## 4. 使用建议

- 后续创建 issue 时，优先关联到对应 milestone
- 每个功能 plan 建议补充 `所属 Milestone: Mx`
- milestone 的关闭应以阶段退出条件满足为准，而不是以 issue 数量清零为准

## 5. 维护注意事项

- 若未来要调整阶段定义，应先更新本地 [project-milestones.md](/Users/andyhao/workspace/p4home/docs/project-milestones.md)，再同步 GitHub
- 若未来需要更细粒度管理，可以在 milestone 下配合 issue labels 或 GitHub Projects 使用
